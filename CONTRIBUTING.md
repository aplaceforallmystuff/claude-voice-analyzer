# Contributing to Claude Voice Analyzer

Thanks for your interest in improving the voice-analyzer skill!

## Ways to Contribute

### Improve Analysis Dimensions

The skill examines six dimensions of voice. You can help by:

- Adding new sub-dimensions within each category
- Refining the questions asked during analysis
- Adding examples of patterns to look for

### Enhance Output Format

The VOICE.md template can be improved:

- Better organization of sections
- More actionable guidance
- Clearer examples

### Add Voice Examples

Help others understand voice analysis:

- Anonymized examples of different voice types
- Before/after comparisons
- Edge cases and how to handle them

### Expand Sample Guidance

Help users select better samples:

- Additional "good sample" criteria
- More alternative sources
- Industry-specific guidance

## Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b improve-analysis`)
3. Make your changes to `skills/voice-analyzer/SKILL.md`
4. Test with various writing samples
5. Submit a pull request

## Guidelines

### Keep It Practical

Voice analysis should be:
- Completable in 15-30 minutes
- Actionable (not just descriptive)
- Testable with the validation prompts

### Maintain Portability

The VOICE.md output should:
- Work across different AI tools
- Be easy to update over time
- Not require external dependencies

### Respect Privacy

When contributing examples:
- Use anonymized or fictional samples
- Don't include identifying information
- Get permission if using real writing

## Questions?

Open an issue for discussion before major changes.
